# Wearable biosignal comparison pipeline 

This repository contains the full preprocessing and visualization pipeline used for comparing three wearable devices (Shimmer3 GSR+, and EmotiBit) across two experimental phases. The code corresponds to the following methods:

- Signal preprocessing (outlier removal, filtering, resampling)
- Synchronization across devices
- Extraction of PPG/BVP, GSR/EDA, ACC and derived signal HR
- Generation of the correlation matrices and plots, and time-course visualizations

> **Raw data are not included** in this GitHub repository, but they are publicly available on Zenodo (see DOI below).
> The dataset expected directory structure and data formats are documented below to ensure full reproducibility.
> **Important**: Only the second dataset (Scenario 2 on Zenodo) is used.

[![DOI](https://zenodo.org/badge/DOI/10.5281/zenodo.19218368.svg)](https://doi.org/10.5281/zenodo.19218368)

## 🧩 Repository structure
```bash
preprocessing.ipynb # Preprocessing pipeline (outlier detection, filtering, resampling...)
visualisation_final.ipynb # Visualizations (signal traces, correlations...)
requirements.txt # Python dependencies

utils/
├── filtering.py 
├── loader.py 
├── metrics.py 
└── outliers.py 
```
The repository originally contained a `data/` folder with all recordings, but **these files are not distributed**.  
Instead, we document the expected structure so users can reproduce the workflow with their own data.


## 📁 Expected `data/` directory

The pipeline assumes the following directory structure (`data/second` on Zenodo):

```bash
data/
├── emotibit_dorsal/           # EmotiBit (dorsal wrist placement)
├── emotibit_volar/            # EmotiBit (volar wrist placement)
├── shimmer_wrist/             # Shimmer3 GSR+ at the wrist (GSR volar, BVP/PPG dorsal)
├── shimmer_fingers/           # Shimmer3 GSR+ at the fingers
└── Stamps/                    # Event/stimulus markers

```

Each device folder contains:

- **EmotiBit**
  - One folder per participant containing the following:  
    - `raw/` contains the original EmotiBit export (`.csv` + `_info.json`)
    - `parsed/`[^1]  contains one file per channel (AX, AY, AZ, GX, HR, PG, etc.)
    - Includes synchronization files: `timeSyncMap.csv`, `timesyncs.csv`
    ```bash
    EmotiBit/
    ├── <participant_id>
    │   ├── raw/                      # File .csv + .json original from EmotiBit
    │   │   ├── <timestamp>.csv
    │   │   └── <timestamp>_info.json
    │   └── parsed/                  # Each channel
    │       ├── *_AX.csv             
    │       ├── *_AY.csv
    │       ├── *_AZ.csv
    │       ├── ... (rest of the channels)
    │       ├── *timeSyncMap.csv     # Synchronisation map
    │       ├── *timesyncs.csv       # Detected synchronisations
    │       └── *_info.json          # Processed metadata
    ├── ... (rest of participants)
    ```
[^1]: EmotiBit recordings follow the structure generated by the official [EmotiBit DataParser tool](https://github.com/EmotiBit/EmotiBit_Docs/blob/master/Working_with_emotibit_data.md#emotibit-dataparser).

  
- **Shimmer3 GSR+**
  - One CSV per participant (GSR, PPG, ACC and metadata already combined)
  ```bash
  Shimmer/
  └── <participant_id>.csv 
  ```

- **Stamps**
  - Event/stimulus marker files (`.txt`) aligned with each recording
  ```bash
  Stamps/
  └── <participant_id>.txt
  ```


You can replicate the analysis by reproducing this directory structure with your own recordings.


## 🔧 Installation
```bash
pip install -r requirements.txt
```
Recommended Python version: 3.10+


## ▶ How to run the analysis

1. Place your data following the structure described above.
2. Install requirements.txt
3. Run `preprocessing.ipynb`
4. Run `visualisation.ipynb`
